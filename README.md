# EDT Tracing

Плагин для [1C:EDT](https://edt.1c.ru/), добавляющий пошаговую трассировку выполнения кода с возможностью фильтрации модулей и экспорта результатов.

## Возможности

- **Пошаговая трассировка** — последовательная запись всех выполняемых строк BSL-кода по всем debug-таргетам
- **Фильтрация модулей** — исключение из трассировки модулей по шаблону
- **Экспорт** — CSV и JSONL

## Установка

- **Через GitHub:** Справка → Установить новое ПО → Добавить → `https://timkonzt.github.io/EDT-Tracing/`
- **Из ZIP-архива:** Справка → Установить новое ПО → Добавить → Архив... → указать `edt-tracing-plugin_<версия>.zip` из [релизов](https://github.com/TimkoNZT/EDT-Tracing/releases)
  
  

## Использование

1. Запустить отладку \*
2. Нажать кнопку **Трассировка** на панели Отладка (или `Запуск → Трассировка`)
3. Остановить трассировку повторным нажатием или дождаться окончания выполнения
4. Двойной клик по записи — открыть модуль на нужной строке
5. Кнопки JSON или CSV для экспорта

\* п.1 и п.2 можно выполнять в любом порядке

## Фильтр модулей

Код модуля или метода попадающего под фильтр не записывается в трассировку.

Фильтр проверяется по образцу `Тип.ИмяМодуля.Модуль.ИмяМетода` (без аргументов):

- `ОбщийМодуль.Пользователи.Модуль.АвторизованныйПользователь` — точное совпадение по имени метода
- `ОбщийМодуль.Пользователи.*` — любой метод модуля Пользователи
- `ОбщийМодуль.ОбщегоНазначения*` — любой из модулей с префиксом ОбщегоНазначения
